2. Explain in your own words what are weak and strong qualifiers for properties and
Variables.


Weak qualifiers: 
Weak qualifiers used to prevent the retain cycle.Weak qualifiers shows that object don’t want to take ownership of another object 
This weak objects will be in memory until the strong objects keeps reference to it and until then the retail count does not get increased and it get destroyed if object no longer keep reference.
In short it is like keep this as long as someone else points to it strongly


Strong qualifiers:
Strong qualifiers means that you want to own the object you are referencing and Whenever an object keeps strong reference to another object the retain count of that object is increased by 1
All properties are ‘strong’ by default.
Object remains alive until there is strong points to it.


        
3. Explain what does it mean to declare a property weak. What are the dangers of using a
property / variable declared as weak?
Declaring a property weak means that the property does not owns the object and its the reference count is not get increased its points until another object keep reference if no object keep reference then reference automatically become nil .


Dangers of using a property/ variable declared as weak is that object can unexpectedly disappear if no strong reference exist and this can cause issue because it becomes nil so program can crash if we have not written logic to manage nil values.


        
4. Explain in your own words what is a retain cycle
When two objects referring to each other and both are strong reference and due to this reason each others reference count will never get drop to zero causing memory wastage this is known as retain cycle.
 Eg: 
Employee (Strong) —--> Company
Company (Strong) —-> Employee
Here both objects strongly reference each other and because of this memory is never released.


To solve the retail cycles one side should be declared as weak and other is strong
